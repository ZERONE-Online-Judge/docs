# 대회 참가 로그인 플로우

## 목적

참가 팀이 대회별 정책에 맞는 방식으로 로그인하고 팀 워크스페이스에 진입하는 흐름을 정의한다.

## 기본 플로우

1. 사용자가 대회 홈에서 참가 로그인을 선택한다.
2. 시스템은 대회에 등록된 참가자 이메일 입력을 요구한다.
3. 입력 값이 유효한 참가자 이메일이라면 해당 이메일로 OTP를 발송한다.
4. 사용자가 OTP를 입력한다.
5. 인증 성공 시 팀 워크스페이스로 이동한다.
6. 팀 상태가 `invited`라면 최초 로그인 성공 시 `active`로 전환한다.

## 인증 정책

- 참가자 인증은 등록된 참가자 이메일과 이메일 기반 OTP를 기준으로 한다.
- OTP 수신지는 로그인에 사용한 참가자 이메일이다.
- OTP 유효 시간은 `10분`이다.
- OTP 발송 요청은 같은 참가자 이메일 기준으로 제한한다.
- OTP 검증 실패는 제한 대상이며, 반복 실패 시 일정 시간 재시도를 차단할 수 있어야 한다.
- OTP 원문은 저장하지 않고 해시한 값만 저장해야 한다.
- OTP 발송/성공/실패/만료/차단 이벤트는 추적 가능해야 한다.
- 이메일 발송 남용을 막기 위해 IP 기준 rate limit도 함께 적용할 수 있어야 한다.

## 세션 정책

- 참가자 세션 검증 단위는 팀 전체가 아니라 참가자 이메일 단위다.
- 동시 접속 제한도 참가자 이메일 단위로 관리한다.
- 같은 브라우저의 여러 탭 사용은 허용한다.
- 같은 참가자가 다른 기기에서 로그인하려고 하면 기존 기기 연결 해제 후 새 기기에서 계속할지, 새 로그인을 취소할지 선택하게 한다.
- 다른 기기 로그인으로 기존 세션을 해제한 경우 기존 기기의 refresh token은 무효화되어야 한다.
- 참가자 인증 토큰은 JWT 기반으로 발급한다.
- refresh token은 서버 DB에 해시 형태로 저장하고, 기기 세션과 연결한다.
- access token은 짧게 유지하고, refresh token은 대회 종료 시각 또는 운영 정책에 따라 만료되어야 한다.

## 대회 정책 반영

- 대회 참가 단위는 `n인 1팀`이며 `n >= 1`이다.
- 대회 운영진은 팀 이름, 팀장 이름/메일, 팀원 이름/메일을 등록한다.
- 등록된 팀장/팀원 이메일만 참가자 로그인을 할 수 있다.
- 팀 구성과 참가 자격은 대회 등록 데이터와 대회 운영자 확인 절차를 기준으로 검증한다.

## 로그인 후 보여줘야 하는 것

- 팀명
- 현재 대회명
- 참가자 이메일 세션 안내
- 문제 목록
- 제출 현황
- 공지/질문 게시판 링크

## 예외 처리

- 미등록 이메일
- 로그인 허용 대상이 아닌 참가자 이메일
- 만료된 OTP
- 잘못된 OTP
- OTP 요청 제한 초과
- OTP 검증 실패 제한 초과
- 다른 기기에서 이미 로그인된 세션
- 비활성/실격 팀
