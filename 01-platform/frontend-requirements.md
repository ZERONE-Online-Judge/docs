# 프론트 요구사항

## 기술 기준

- `Vite` 기반 `React`
- 스타일링: `Tailwind CSS`
- 서버 상태 관리: `TanStack Query`
- 클라이언트 전역 상태 관리: `Zustand`
- 신규 요구사항과 정책을 기준으로 새 디자인 설계
- 공개 영역, 참가 팀 영역, 운영 영역을 명확히 구분

## 필수 화면

### 공통

- 메인 페이지
- 대회 목록 페이지
- 대회 홈 페이지
- 권한자 로그인 페이지
- 비밀번호 최초 설정 / 재설정 페이지
- 대회 규정집 페이지
- 공지사항 게시판 / 질문 게시판
- 서비스 공지 게시판
- 공개 채점 서버 현황 페이지

### 대회참가자용

- 대회 참가 로그인 페이지
- 문제 목록 페이지
- 문제 상세 페이지
- 제출 페이지
- 제출 현황 페이지
- 제출 상세 페이지

### 운영자/운영매니저용

- 운영자 대시보드
- 참가 팀 관리 페이지
- 스코어보드 운영 페이지
- 공지사항 운영 페이지
- 질문 게시판 운영 페이지
- 서비스 공지 게시판 운영 페이지
- 서비스 마스터용 대회/서비스 관리 페이지

### 출제자용 화면 분류

- 문제 관리 페이지
- 문제 본문 편집 페이지
- 예제 입출력 관리 페이지
- 테스트케이스/리소스 관리 페이지
- 서브태스크/배점 관리 페이지

출제자용 화면은 현재 기준으로는 별도 독립 계정 역할이라기보다, 문제 관리 권한을 가진 대회 운영자 또는 대회 운영매니저가 접근하는 화면으로 본다.
향후 `출제자`를 독립 역할로 분리할 경우 권한 문서와 함께 재정의해야 한다.

## 프론트 공통 요구사항

- 로그인 상태와 역할에 따라 메뉴를 다르게 노출해야 한다.
- 대회 상태에 따라 노출 기능과 CTA가 달라져야 한다.
- 참가 팀 플로우는 짧고 단순해야 한다.
- 운영 화면은 상태 확인과 변경 이력이 분명해야 한다.
- 채점 상태는 long polling 기반 준실시간 갱신으로 시작한다.
- 긴급 공지는 상단 배너에서 일부만 노출될 수 있으므로, 항상 공지사항 상세 또는 목록으로 이동하는 CTA를 제공해야 한다.
- 사용자 또는 운영자가 글로 작성하는 모든 콘텐츠는 서식 기능을 지원해야 한다.
- 최소한 `Markdown` 기반 서식 중 `h1`, `h2`, `h3`, `h4`, 기울임, 굵게, 표, 이미지 문법을 지원해야 한다.
- 제목 내부에서도 기울임과 굵게 강조 문법을 허용할 수 있어야 한다.
- 수식용 `LaTeX` 문법 입력을 지원해야 한다.
- 수식 렌더링은 `KaTeX` 등 클라이언트 렌더러를 통해 일관되게 표시할 수 있어야 한다.
- 이미지는 업로드 후 `Markdown` 이미지 문법으로 참조할 수 있어야 한다.
- 이미지 업로드는 `jpg`, `jpeg`, `png`, `webp` 확장자만 허용하고 파일당 최대 `512MB`로 제한한다.
- 외부 URL 이미지는 허용하지 않는다.
- 이미지 렌더링은 실제 파일 경로 직접 노출보다 내부 asset 참조 기반으로 처리하는 것이 바람직하다.
- HTML 직접 입력은 허용하지 않아야 한다.
- 렌더링 과정에서 XSS를 방지할 수 있도록 입력 정제와 안전한 렌더링 규칙이 필요하다.

## 사용자 경험 요구사항

- 메인 페이지에서 서비스 목적과 대회 운영 방식을 바로 이해할 수 있어야 한다.
- 대회 목록에서 현재 진행/예정/종료 대회를 구분할 수 있어야 한다.
- 참가 로그인 페이지는 참가자 이메일 기반 OTP 로그인임을 명확히 보여줘야 한다.
- 운영자 화면은 대회 시작 전/중/후 상태에 따라 허용 범위를 명확히 안내해야 한다.
- 긴급 공지 배너의 요약 노출만으로 핵심 내용이 누락되지 않게, 원문 확인 진입이 명확해야 한다.
