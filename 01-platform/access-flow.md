# 접속 플로우

## 목적

이 문서는 사용자 유형별로 서비스에 어떻게 진입하고, 어떤 로그인 흐름으로 이동하는지 정리한다.

## 접속 대상

- 비로그인 사용자
- 참가 팀 사용자
- 서비스 마스터
- 서비스 매니저
- 대회 운영자
- 대회 운영매니저

## 상위 진입 플로우

1. 사용자는 메인 페이지에 진입한다.
2. 메인 페이지에서 대회 목록, 운영 규정, 공개 채점 서버 현황으로 이동할 수 있어야 한다.
3. 특정 대회를 고르면 대회 홈으로 이동한다.
4. 대회 홈에서 문제 접근 가능 여부, 로그인 필요 여부, 게시판 접근 가능 여부가 대회 상태에 따라 결정된다.

## 참가 팀 접속 플로우

1. 사용자는 대회 목록 또는 대회 홈에서 참가 로그인을 선택한다.
2. 시스템은 대회에 등록된 참가자 이메일 입력을 요구한다.
3. 해당 이메일로 발송된 OTP 인증을 통과하면 팀 워크스페이스로 진입한다.
4. 팀 워크스페이스에서 문제 목록, 제출 현황, 스코어보드, 공지, 질문 게시판으로 이동한다.

## 권한자 접속 플로우

1. 사용자는 권한자 로그인 페이지로 이동한다.
2. 이메일과 비밀번호로 로그인한다.
3. 로그인 후 자신의 전역 권한과 대회 단위 권한을 기준으로 접근 가능한 메뉴를 노출한다.
4. 서비스 마스터/서비스 매니저는 서비스 관리 영역으로, 대회 운영자/매니저는 담당 대회 운영 영역으로 진입한다.
5. 서비스 마스터는 개별 대회 운영 영역에 직접 진입할 수 있고, 서비스 매니저는 permission과 scope가 허용하는 대회 운영 영역에만 진입할 수 있다.

## 접속 정책 핵심

- 참가 팀 로그인과 권한자 로그인은 분리된 화면과 정책을 가져야 한다.
- 대회 참가 단위는 항상 `n인 1팀`이며 `n >= 1`이다.
- 대회 운영진은 팀 이름, 팀장 이름/메일, 팀원 이름/메일을 등록한다.
- 참가자 로그인 식별자는 등록된 참가자 이메일이다.
- OTP는 참가자 이메일별로 발송한다.
- 로그인 세션과 동시 접속 제한은 팀 단위가 아니라 참가자 이메일 단위로 관리한다.
- 권한 없는 화면은 메뉴와 API에서 모두 차단해야 한다.
