# 서비스 공지 게시판

## 목적

서비스 공지 게시판은 특정 대회에 속하지 않는 플랫폼 단위 공지를 제공한다.
서비스 운영 정책, 점검, 장애, 기능 변경, 대회 개최 안내 같은 전역 메시지를 관리자가 작성해 공지하는 용도다.
일반 이용자가 글을 작성하거나 답글을 다는 게시판 기능은 제공하지 않는다.

## 공지 유형

- 일반 공지
- 서비스 점검 공지
- 장애/복구 공지
- 정책 변경 공지
- 기능 변경 공지
- 대회 개최 안내
- 긴급 공지

## 공개 범위

- 기본적으로 비로그인 사용자도 목록과 상세를 볼 수 있다.
- 숨김 또는 비공개 처리된 공지는 서비스 마스터와 권한 있는 서비스 매니저만 볼 수 있다.
- 긴급 공지는 메인/루트 페이지와 상단 네비게이션 배너에 노출할 수 있다.
- 긴급 공지는 최근 1개만 노출한다.
- 대회 페이지에서는 대회 긴급 공지가 서비스 긴급 공지보다 우선한다.
- 긴급 공지 배너는 요약 노출용이며, 원문은 반드시 공지 상세 페이지에서 확인할 수 있어야 한다.

## 필수 기능

- 공지 목록 조회
- 공지 상세 조회
- 공지 작성
- 공지 수정
- 공지 삭제 또는 숨김
- 공지 검색
- 상단 고정
- 긴급 노출
- 공개 종료
- 공개/비공개 전환
- Markdown 본문 작성
- 이미지 업로드 및 본문 참조

## 관리 권한

- 서비스 마스터는 모든 서비스 공지 게시판 작업을 수행할 수 있다.
- 서비스 매니저는 `service.notice.manage` 권한이 있을 때 공지 작성/수정/삭제/숨김을 수행할 수 있다.
- 서비스 매니저는 `service.notice.emergency_publish` 권한이 있을 때 긴급 공지 노출을 설정하거나 해제할 수 있다.
- `service.notice.view_private` 권한이 있으면 비공개 또는 숨김 처리된 공지를 조회할 수 있다.
- 대회 운영자/운영매니저 권한만으로는 서비스 공지 게시판을 관리할 수 없다.

## 대회별 공지와의 구분

- 서비스 공지는 특정 `contest_id`를 필수로 갖지 않는다.
- 대회별 공지는 대회 홈과 대회 워크스페이스 안에서 노출된다.
- 서비스 긴급 공지는 모든 공개 페이지 또는 운영 정책에 따른 전역 페이지에 노출될 수 있다.
- 대회별 긴급 공지는 해당 대회 페이지 안에서만 노출한다.

## 본문 정책

- 본문은 Markdown 기반 서식을 지원한다.
- 최소한 `h1`, `h2`, `h3`, `h4`, 기울임, 굵게, 표, 이미지 문법을 지원한다.
- 수식용 `LaTeX` 문법 입력을 지원할 수 있다.
- 렌더링은 `KaTeX` 등으로 일관되게 표시할 수 있어야 한다.
- 이미지는 `jpg`, `jpeg`, `png`, `webp` 확장자만 허용하고 파일당 최대 `512MB`로 제한한다.
- 외부 URL 이미지는 허용하지 않는다.
- HTML 직접 입력은 허용하지 않는다.
- 렌더링 과정에서 XSS를 방지하기 위해 입력 정제와 안전한 렌더링 규칙을 적용한다.

## 감사 로그

아래 작업은 감사 로그 대상이다.

- 공지 작성
- 공지 수정
- 공지 삭제 또는 숨김
- 긴급 노출 설정/해제
- 상단 고정 설정/해제
- 공개 범위 변경
- 공개 종료 시각 변경

## 관련 공개 페이지

- 서비스 공지 목록
- 서비스 공지 상세
- 메인 페이지의 최신 공지 요약
- 긴급 공지 배너 또는 상단 고정 영역
