# 대회 운영자 문제 관리 페이지 상세

이 문서는 문제 등록 페이지와 문제 관련 설정 화면을 탭 단위로 정리한 상세 초안이다.
현재 문서에 이미 흩어져 있는 요구사항을 페이지 구조 기준으로 재배치한다.

## 목적

- 문제 등록/수정 화면에서 어떤 탭이 필요한지 정리한다.
- 문제 본문, 예제, 리소스, 테스트케이스, 배점 설정을 한 페이지 구조 안에서 본다.
- 향후 프론트 화면 설계와 백엔드 API 분리를 위한 기준 초안으로 사용한다.

## 페이지 범위

- 문제 목록 조회
- 문제 생성
- 문제 수정
- 문제 삭제
- 문제 순서 변경
- 문제별 세부 설정 관리

## 페이지 공통 구성

### 좌측 또는 상단 목록 영역

- 문제 목록
- 문제 코드
- 문제 제목
- 표시 순서
- 현재 편집 가능 여부

### 우측 또는 본문 작업 영역

- 선택한 문제의 세부 탭
- 저장 / 취소 / 삭제 액션
- 시작 전 / 진행 중 / 종료 후 상태 안내
- 수정 제한 안내 문구

### 공통 상단 액션

- 새 문제 추가
- 문제 저장
- 문제 삭제
- 문제 순서 변경
- 이전/다음 문제 이동

## 탭 구성 초안

### 1. 기본 정보 탭

목적:

- 문제의 식별 정보와 기본 메타데이터를 입력한다.

포함 항목:

- 문제 코드
- 제목
- 표시 순서
- 시간 제한
- 메모리 제한

관련 규칙:

- 문제 코드는 대회 내부에서 식별 가능해야 한다.
- 표시 순서는 문제 목록과 스코어보드 노출 순서에 영향을 줄 수 있다.

### 2. 문제 본문 탭

목적:

- 참가자가 실제로 읽는 문제 설명을 편집한다.

포함 항목:

- 본문
- 입력 설명
- 출력 설명
- 제약 조건
- 서식 미리보기
- 수식 렌더링 미리보기

관련 규칙:

- 대회 진행 중에는 문제 본문 수정이 금지된다.
- 운영자가 보는 범위와 참가자가 보는 범위는 분리되어야 한다.
- 문제 본문 작성 영역은 `Markdown` 기반 서식 중 `h1`, `h2`, `h3`, `h4`, 기울임, 굵게, 표, 이미지 문법을 지원해야 한다.
- 제목 내부에서도 기울임과 굵게 강조 문법을 허용할 수 있어야 한다.
- 수식용 `LaTeX` 문법 입력을 지원해야 한다.
- 본문 렌더링은 `KaTeX` 등으로 일관되게 표시할 수 있어야 한다.
- 이미지는 업로드 후 `Markdown` 이미지 문법으로 참조할 수 있어야 한다.
- HTML 직접 입력은 허용하지 않아야 한다.
- 렌더링 과정에서 XSS를 방지할 수 있도록 입력 정제와 안전한 렌더링 규칙이 필요하다.

### 3. 예제 입출력 탭

목적:

- 참가자에게 공개되는 예제 입력/출력을 관리한다.

포함 항목:

- 예제 입력
- 예제 출력
- 예제 추가 / 수정 / 삭제

관련 규칙:

- 예제 입출력은 문제 본문과 함께 대회 진행 중 수정 금지 범주에 포함된다.

### 4. 리소스 탭

목적:

- 문제 본문에 연결되는 이미지 등 리소스를 관리한다.

포함 항목:

- 리소스 업로드
- 리소스 삭제
- 리소스 목록
- 리소스 참조 정보

관련 규칙:

- 본문용 이미지 파일만 허용한다.
- 최소 허용 확장자는 `png`, `jpg`, `jpeg`, `webp`다.
- 파일당 최대 크기는 `512MB`다.
- 외부 URL 이미지는 허용하지 않고, 업로드된 내부 asset만 참조할 수 있다.
- 업로드 이미지는 사용자 파일명이 아니라 내부 `asset_id` 기반으로 저장하는 것이 바람직하다.
- 저장 경로는 `contest/{contest_id}/problem/{problem_id}/images/...` 같은 prefix 구조를 가질 수 있어야 한다.
- 본문에서는 실제 파일 경로 대신 내부 참조 식별자 기준으로 `Markdown` 이미지 문법을 사용할 수 있어야 한다.
- 원본 파일명은 메타데이터로만 보관하고, 실제 저장 파일명으로 사용하지 않아야 한다.
- 확장자와 함께 실제 파일 타입 검증이 가능해야 한다.

### 5. 테스트케이스 탭

목적:

- 채점에 사용되는 테스트케이스를 수동 등록하고 관리한다.

포함 항목:

- 테스트케이스 목록
- 테스트케이스 수동 등록
- 테스트케이스 수정
- 테스트케이스 삭제

후속 상세화 필요:

- 입력/출력 데이터 형식
- 공개/비공개 구분 여부
- 그룹 또는 서브태스크 연결 방식

### 6. 테스트케이스 제너레이터 탭

목적:

- 테스트케이스 생성기 기반 관리가 필요한 문제를 지원한다.

포함 항목:

- 제너레이터 목록
- 제너레이터 등록
- 제너레이터 수정
- 제너레이터 삭제

후속 상세화 필요:

- 생성기 실행 정책
- 버전 관리 방식
- 생성 결과와 테스트케이스 확정 절차

### 7. 채점 / 배점 탭

목적:

- 문제의 채점 제도별 설정을 관리한다.

공통 항목:

- 채점 제도별 점수 설정

AC/WA 대회:

- 부분 점수 미사용
- `Accepted`만 정답 처리
- 나머지 최종 판정 결과는 오답 및 페널티 가산 처리

점수제 대회:

- 문제 만점 `100점`
- 일반문제 또는 서브태스크형 문제 구분
- 서브태스크 또는 기준 목록
- 기준별 테스트케이스 묶음
- 기준별 배점
- 배점 합계 검증 결과

후속 상세화 필요:

- 일반문제와 서브태스크형 문제를 화면에서 어떻게 구분 표시할지
- 기준 추가/삭제 UI
- 테스트케이스와 기준의 연결 방식
- 배점 합계 오류 표시 방식

문제 유형 판별 기준:

- 배점 구간이 `100점` 단일 하나뿐이면 일반문제로 본다.
- 최대치 `100점`을 포함하면서 `100점 미만`의 중간 배점 기준이 하나라도 있으면 서브태스크형 문제로 본다.
- 점수제 대회의 일반문제는 `0점 또는 100점`만 나오는 문제다.
- 각 기준은 출제자가 지정한다.
- 각 기준은 연결된 모든 테스트케이스가 정답 처리되어야 통과한 것으로 본다.
- 테스트케이스 진행 중 오답 처리가 발생하면 채점기는 이후 케이스를 진행하지 않고 즉시 해당 제출을 오답으로 확정한다.

## 페이지 상태별 제약

### 시작 전

- 문제 생성 가능
- 문제 수정 가능
- 예제/리소스/테스트케이스/배점 설정 가능

### 진행 중

- 문제 본문 및 데이터 수정 금지
- 문제 배점 규칙 및 채점 제도 변경 금지

### 종료 후

- 결과 검토 목적의 조회 가능
- 재사용 또는 공개 정책에 따른 후속 작업 가능
