# 대회 운영자 페이지 참가자 추가 삭제 변경

## 주요 작업

- 참가 팀 추가
- 참가 팀 수정
- 참가 팀 삭제
- 참가 상태 변경
- 실격 처리
- 팀장 이름/메일 추가 또는 변경
- 팀원 이름/메일 추가 또는 변경
- 참가자 이메일별 로그인 상태 확인

## 등록 데이터

- 팀명
- 팀 인원 수 `n`
- 팀장 이름
- 팀장 이메일
- 팀원 이름 목록
- 팀원 이메일 목록
- 참가 상태
- 참가자 이메일별 세션 상태

## 규칙

- 참가 단위는 항상 `n인 1팀`이며 `n >= 1`
- 참가자 로그인 식별자는 등록된 참가자 이메일
- OTP는 참가자 이메일별로 발송
- 로그인 세션과 동시 접속 제한은 팀 단위가 아니라 참가자 이메일 단위로 관리
- 운영자/운영매니저는 같은 대회의 참가 팀으로 등록할 수 없음

## 운영 상태

- `invited`: 초대 또는 등록은 완료됐지만 아직 최초 로그인하지 않은 상태
- `active`: 최초 로그인 후 정상 참가 가능한 상태
- `disqualified`: 실격 처리된 상태
- `disabled`: 운영자가 비활성화한 상태

## 상태별 기본 동작

| 상태 | 로그인 | 문제 조회 | 제출 | 스코어보드 노출 |
| --- | --- | --- | --- | --- |
| `invited` | 가능 | 최초 로그인 성공 후 `active` 전환 뒤 가능 | 최초 로그인 성공 후 `active` 전환 뒤 가능 | 대회 정책에 따름 |
| `active` | 가능 | 가능 | 가능 | 노출 |
| `disqualified` | 불가 | 불가 | 불가 | 대회 정책에 따라 숨김 또는 실격 표시 |
| `disabled` | 불가 | 불가 | 불가 | 숨김 |

실격 팀을 스코어보드에 숨길지, 실격 상태로 표시할지는 대회별 공개 정책으로 결정해야 한다.
`invited` 상태의 팀이 최초 로그인에 성공하면 `active`로 전환한다.

## 운영 원칙

- 운영 상태와 로그인 이력은 분리해 관리한다.
- 참가자 정보 변경은 감사 로그 대상이다.
- 실격 처리는 별도 사유 기록이 필요하다.
