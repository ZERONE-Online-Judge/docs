# 대회 운영자 페이지 대회 시작 / 중 / 끝 가능한 범위및 규정

## 상태 흐름

대회 상태는 아래 흐름을 기준으로 검토한다.

```text
draft -> scheduled -> open -> running -> ended -> finalized -> archived
```

스코어보드 프리즈는 별도 상태가 아니라 `running` 중 적용되는 스코어보드 공개 정책으로 본다.

## 시작 전

가능:

- 대회 기본 정보 수정
- 참가 팀 등록/수정
- 문제 등록/수정/정렬
- 규정집 작성
- 공지 작성

주의:

- 공개 시각, 시작 시각, 종료 시각 최종 검토 필요
- 참가자 이메일 기반 로그인과 이메일별 세션 정책 확인 필요
- 채점 제도와 동점 처리 규칙 확정 필요

## 진행 중

가능:

- 공지/긴급 공지 운영
- 질문 답변
- 참가 팀 상태 관리
- 제출/스코어보드/큐 모니터링
- 실격 처리

제한:

- 문제 본문 및 데이터 수정 금지
- 테스트케이스 active 버전 변경 금지
- 문제 배점 규칙 및 채점 제도 변경 금지
- 대회 운영자/운영매니저의 재채점 요청 금지
- 공정성에 영향을 주는 설정 변경 금지

## 종료 후

가능:

- 종료 처리
- 결과 검토
- 재채점 결과 검토
- 팀/문제별 점수 개별 조정
- 최종 스코어보드 공개
- 종료 후 문제 공개 정책 집행

원칙:

- 재채점 결과는 대회 당시 스코어보드를 자동 변경하지 않는다.
- 종료 후 수동 점수 조정도 대회 당시 스코어보드를 자동 변경하지 않는다.
- 최종 결과에 반영할 재채점 결과는 권한 있는 서비스 관리자 또는 채점 인프라 권한자가 검토 후 확정해야 한다.
- 최종 결과에 반영할 수동 점수 조정은 조정 사유와 변경 전후 값을 남긴 뒤 운영자가 명시적으로 확정해야 한다.
