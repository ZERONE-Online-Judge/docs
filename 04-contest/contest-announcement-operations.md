# 대회 공지사항 운영

## 공지 유형

- 일반 공지
- 규정 변경 공지
- 운영 공지
- 긴급 공지

## 필수 기능

- 공지 작성
- 공지 수정
- 공지 삭제
- 상단 고정
- 긴급 노출
- 노출 범위 설정
- 본문 서식 지원

## 긴급 공지 운영

- 페이지 상단 네비게이션 바에 강조 형태로 노출
- 사용자가 직접 닫을 수 있어야 함
- 대회별 노출 범위 관리 필요
- 종료 시 자동 숨김 또는 수동 종료 필요
- 긴급 공지는 최근 1개만 노출한다.
- 대회 긴급 공지가 서비스 긴급 공지보다 우선한다.
- 긴급 공지 내용이 길 경우 네비게이션 바 배너에서는 일부만 노출될 수 있음을 전제로 해야 한다.
- 긴급 공지 배너에는 항상 해당 공지사항 상세 또는 공지사항 목록으로 이동하는 진입 기능이 있어야 한다.
- 긴급 공지 전문을 읽어야 하는 사용자를 위해 `자세히 보기`, `공지사항으로 이동` 같은 명확한 CTA가 필요하다.
- 잘린 미리보기 텍스트와 실제 공지 본문이 충돌하지 않도록, 배너는 요약 노출용이고 공지사항 페이지가 원문 기준이 되어야 한다.

## 운영 원칙

- 공지 작성/수정/삭제 권한은 역할에 따라 제어한다.
- 규정 변경 공지는 감사 로그 대상이다.
- 긴급 공지는 일반 공지와 다른 표시 규칙을 가져야 한다.
- 긴급 공지의 원문 보존과 상세 확인 경로는 반드시 공지사항 게시판과 연결되어야 한다.
- 예약 공지는 지원하지 않는다.
- 공지사항과 긴급 공지 본문은 `Markdown` 기반 서식 중 `h1`, `h2`, `h3`, `h4`, 기울임, 굵게, 표, 이미지 문법을 지원해야 한다.
- 제목 내부에서도 기울임과 굵게 강조 문법을 허용할 수 있어야 한다.
- 수식용 `LaTeX` 문법 입력을 지원해야 한다.
- 공지사항과 긴급 공지의 수식 렌더링은 `KaTeX` 등으로 일관되게 표시할 수 있어야 한다.
- 이미지는 업로드 후 `Markdown` 이미지 문법으로 참조할 수 있어야 한다.
- 이미지는 `jpg`, `jpeg`, `png`, `webp` 확장자만 허용하고 파일당 최대 `512MB`로 제한한다.
- 외부 URL 이미지는 허용하지 않는다.
- HTML 직접 입력은 허용하지 않아야 한다.
- 렌더링 과정에서 XSS를 방지할 수 있도록 입력 정제와 안전한 렌더링 규칙이 필요하다.
