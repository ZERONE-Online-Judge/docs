# 대회 게시판 운영

## 범위

이 문서는 질문 게시판과 일반 대회 게시판 성격의 운영 흐름을 다룬다.

## 질문 게시판 기능

- 참가 팀 질문 등록
- 질문 목록 조회
- 질문 상세 조회
- 대회 운영자 또는 권한 있는 대회 운영매니저 답변
- 질문 공개/비공개 설정
- 댓글 공개/비공개 설정
- 질문 본문과 답변 본문 서식 지원

## 운영 원칙

- 질문은 대회 단위로 분리되어야 한다.
- 문제 관련 질문은 문제 식별자와 연결 가능해야 한다.
- 권한에 따라 작성/답변/수정/숨김 범위가 달라져야 한다.
- 참가 팀은 질문 작성 시 공개 또는 비공개를 선택할 수 있다.
- 공개 질문은 제목과 내용이 모든 사용자에게 보인다.
- 비공개 질문은 목록에서 제목과 내용이 모두 비공개로 표시되고, 작성 팀 이외의 참가자는 상세에 접근할 수 없다.
- 대회 운영자와 권한 있는 대회 운영매니저는 공개/비공개 질문을 모두 볼 수 있다.
- 대회 운영자는 비공개 질문을 공개 질문으로 전환할 수 없다.
- 운영자 화면에서는 질문이 공개로 작성됐는지 비공개로 작성됐는지 표시해야 한다.
- 댓글 작성자는 댓글마다 공개 또는 비공개를 선택할 수 있다.
- 공개 댓글은 질문 접근 권한이 있는 사용자에게 보인다.
- 비공개 댓글은 댓글 작성자, 질문 작성 팀, 대회 운영자와 권한 있는 대회 운영매니저만 볼 수 있다.
- 운영자 화면에서는 댓글이 공개로 작성됐는지 비공개로 작성됐는지 표시해야 한다.
- 중요한 안내는 공지사항으로 승격할 수 있어야 한다.
- 질문과 답변 본문은 `Markdown` 기반 서식 중 `h1`, `h2`, `h3`, `h4`, 기울임, 굵게, 표, 이미지 문법을 지원해야 한다.
- 제목 내부에서도 기울임과 굵게 강조 문법을 허용할 수 있어야 한다.
- 수식용 `LaTeX` 문법 입력을 지원해야 한다.
- 질문 게시판의 수식 렌더링은 `KaTeX` 등으로 일관되게 표시할 수 있어야 한다.
- 이미지는 업로드 후 `Markdown` 이미지 문법으로 참조할 수 있어야 한다.
- 이미지는 `jpg`, `jpeg`, `png`, `webp` 확장자만 허용하고 파일당 최대 `512MB`로 제한한다.
- 외부 URL 이미지는 허용하지 않는다.
- HTML 직접 입력은 허용하지 않아야 한다.
- 렌더링 과정에서 XSS를 방지할 수 있도록 입력 정제와 안전한 렌더링 규칙이 필요하다.

## 상태 예시

- 접수됨
- 검토 중
- 답변 완료
