# 대회 채점 제도 구분

이 문서는 대회가 어떤 채점 제도를 사용하는지 구분하기 위한 기준 문서다.
기존 규정은 기본적으로 `AC/WA 대회`를 전제로 했고, 서브태스크형 문제 도입을 위해 `점수제 대회`를 추가한다.

## 목적

- 대회별 채점 제도를 명시적으로 구분한다.
- 문제 채점 방식과 스코어보드 정렬 규칙을 분리해서 관리한다.
- AC/WA 대회 규정과 점수제 대회 규정이 한 문서 안에서 뒤섞이지 않게 한다.

## 채점 제도 종류

### 1. AC/WA 대회

- 기존 문서에서 기본값으로 다루던 대회 유형이다.
- 문제는 정답 또는 오답 중심으로 판정한다.
- 정답으로 인정되는 결과는 `Accepted`만이다.
- `Accepted`를 제외한 모든 최종 판정 결과는 스코어보드에서 오답으로 처리한다.
- 페널티 계산에서는 `Compilation Error`를 제외한다.
- 스코어보드는 일반적으로 `푼 문제 수`, `페널티`, `추가 동점 규칙` 기준으로 정렬한다.
- 부분 점수와 서브태스크 배점은 기본적으로 사용하지 않는다.

### 2. 점수제 대회

- 서브태스크형 문제를 도입한 대회 유형이다.
- 모든 문제는 최대 `100점`을 가진다.
- 점수제 대회라고 해서 모든 문제가 서브태스크형 문제일 필요는 없다.
- 점수제 대회의 문제는 `일반문제`와 `서브태스크형 문제`로 나눌 수 있다.
- 모든 문제는 최종적으로 최대 `100점`을 가진다.

#### 점수제 대회의 문제 유형 구분

- `일반문제`
  - 부분 점수가 존재하지 않는 문제다.
  - 배점 구간이 `100점` 단일 하나뿐인 경우로 본다.
  - 점수제 대회에서는 이 문제를 `0점 또는 100점`만 나오는 문제로 해석한다.
- `서브태스크형 문제`
  - 최대치 `100점`을 포함한 `1개 이상의 배점 구간`이 있고, 그 안에 `100점 미만`의 중간 배점 기준이 존재하는 문제다.
  - 즉, 만점 외에 중간 기준이 있고 해당 기준 통과만으로도 일부 점수를 받을 수 있으면 서브태스크형 문제로 본다.

#### 서브태스크형 문제의 기준 통과 규칙

- 각 기준은 출제자가 직접 정의한다.
- 각 기준은 특정 번호까지의 테스트케이스 집합 또는 출제자가 지정한 테스트케이스 집합과 연결된다.
- 어떤 기준에 배정된 모든 테스트케이스가 정답 처리되어야 그 기준을 통과한 것으로 본다.
- 기준을 통과하면 그 기준에 배정된 점수를 얻는다.
- 어떤 기준이 존재하는지와 각 기준의 배점은 문제마다 다를 수 있다.

#### 테스트케이스 진행 규칙

- 테스트케이스 진행 중 오답 처리가 발생하면 채점기는 이후 테스트케이스를 진행하지 않고 즉시 해당 제출을 오답으로 확정한다.
- 중단된 이후 테스트케이스 결과는 해당 제출의 점수 계산에 사용하지 않는다.

## 점수제 대회의 점수 계산 설정

점수제 대회는 대회 관리자 페이지에서 점수 계산 방법을 세분화해 설정할 수 있어야 한다.
아래 규정은 시스템 기본값이며, 대회별 설정으로 일부 항목을 다르게 둘 수 있다.

기본값:

- 스코어보드는 `총점` 내림차순으로 정렬한다.
- `총점`이 같을 경우 그 점수를 먼저 달성한 팀이 상위에 위치한다.
- 점수는 채점이 완료되어 공식 점수로 반영된 즉시 스코어보드에 적용된다.
- 동점 처리에서 어떤 팀이 특정 총점에 먼저 도달했는지는 해당 점수 증가를 만든 제출의 `submitted_at`을 기준으로 판단한다.
- 문제별 최고 점수만 반영한다.
- 같은 문제에서 더 높은 점수를 받은 제출만 점수 달성 시각을 갱신한다.
- 같은 점수 또는 낮은 점수의 제출은 기존 점수와 기존 달성 시각을 유지한다.

대회별 설정 항목:

- 채점 제도 유형: `AC/WA`, `점수제`
- 점수제 문제 유형 허용 범위: 일반문제만 허용, 서브태스크형 포함 허용
- 문제별 배점 합계 검증: 모든 문제 최대 `100점` 검증
- 점수 반영 방식: 채점 완료 후 즉시 공식 점수 반영
- 점수 정렬 기준: 총점 내림차순
- 동점 처리 기준: 기본값은 `submitted_at`
- 중복 순위 허용 여부
- 중복 순위 불허 시 보조 tie-break 또는 중복 순위 허용 전환
- 공개 스코어보드 프리즈 정책

## 점수 달성 시각 규정

- 점수제 대회는 `총점`과 함께 `현재 총점을 처음 달성한 시각`을 함께 관리해야 한다.
- 이 시각은 동점자 처리의 기준값으로 사용된다.
- 기본값에서 `총점 달성 시각`은 해당 총점 도달에 기여한 제출의 `submitted_at`으로 정의한다.
- 채점 완료 시각이나 점수 반영 시각은 스코어보드 갱신에는 사용할 수 있지만, 동점 처리 기준으로 사용하지 않는다.

근거:

- 채점 시간, 채점 대기열, 채점 서버 성능 차이가 순위에 영향을 주지 않게 하기 위함이다.

## 동점 처리 기준까지 같은 경우

아래 경우에는 위 규칙만으로도 완전한 순위 분리가 보장되지 않을 수 있다.

- 두 팀이 정확히 같은 `총점`을 정확히 같은 `submitted_at`에 달성한 경우
- 시각 정밀도가 초 단위처럼 거칠어서 실제로는 순서가 달라도 같은 시각으로 기록되는 경우
- 여러 제출이 같은 제출 시각으로 기록되는 경우

기본 정책:

- 문제별 점수, 풀이 문제 수, 제출 시각 정밀도를 고려하면 이 상황은 매우 드문 예외로 본다.
- 같은 총점, 같은 `submitted_at`까지 모두 같다면 별도 보조 tie-break를 적용하지 않고 중복 순위를 허용한다.

비추천:

- `submission_id`나 팀 등록 번호처럼 경기력과 무관한 내부 식별자로 순위를 강제로 가르는 방식

## 형평성 검토 포인트

- `먼저 달성한 팀 우선`의 기준을 `submitted_at`으로 두면 채점 대기열과 서버 상태가 순위에 개입하는 문제를 줄일 수 있다.
- 다만 채점 결과는 완료 후에 반영되므로, 화면에 점수가 나타나는 시각과 동점 처리 기준 시각이 다를 수 있다.
- 운영 규정과 UI는 이 차이를 참가자가 이해할 수 있게 표시해야 한다.

운영 규정에서 반드시 명확히 할 항목:

- 시각 정밀도가 어느 단위인지
- 정확히 같은 시각까지 동률이면 중복 순위를 허용한다는 점
- 기본값과 다르게 운영하는 대회라면 어떤 계산 정책을 사용하는지

## 추천 방향

- 대회 설정에 `채점 제도 유형`을 명시적으로 둔다.
- 대회 설정에 `점수 계산 정책`을 별도 그룹으로 둔다.
- 점수제 대회에서는 문제별 `서브태스크 구성`과 `배점 합계 100점` 검증 규칙을 둔다.
- 점수제 대회 스코어보드에는 `총점`, `문제별 점수`, `현재 총점 달성 제출 시각`을 함께 저장하거나 projection 할 수 있어야 한다.

## 비추천 방향

- AC/WA 대회 규정 문서 안에 점수제 예외를 덧붙이는 방식
- `부분 점수 허용 여부` 같은 단일 boolean으로 채점 제도를 표현하는 방식
- `총점 동점 시 먼저 달성한 팀 우선`만 적고, 정확히 같은 시각의 처리 규칙을 비워 두는 방식
- 대회별 설정값 없이 모든 대회를 하나의 점수 계산 방식으로 고정하는 방식
