# 대회 구성 요소

## 핵심 구성 요소

- 대회 메타데이터
  - 이름
  - 주최 기관
  - 개요
  - 상태
  - 공개 정책
- 일정
  - 공개 시각
  - 시작 시각
  - 종료 시각
  - 프리즈 시각
  - 최종 결과 확정 시각
- 채점 제도
  - 채점 제도 유형
  - 점수 계산 정책
  - 동점 처리 정책
  - 중복 순위 허용 여부
- 참가 정책
  - 팀 인원 수 `n`
  - 팀장 이름/메일
  - 팀원 이름/메일
  - 참가자 이메일 기반 로그인
  - 참가자 이메일별 세션과 동시 접속 제한
- 인력
  - 운영자
  - 운영매니저
- 콘텐츠
  - 문제
  - 리소스
  - 예제
  - 테스트케이스
- 운영 정보
  - 공지사항
  - 질문 게시판
  - 규정집
- 진행 데이터
  - 제출
  - 팀별 풀이 현황
  - 스코어보드
  - AC/WA 대회의 문제별 풀이 여부와 페널티
  - 점수제 대회의 문제별 점수와 총점

## 필수 원칙

- 대회 단위 데이터 경계가 명확해야 한다.
- 운영 정보와 참가 정보와 채점 정보가 서로 섞이지 않아야 한다.
- 대회 상태에 따라 수정 가능 범위가 달라져야 한다.

## 대회 상태 초안

대회 상태는 백준식 대회 운영을 기준으로 아래 흐름을 우선 검토한다.

```text
draft -> scheduled -> open -> running -> ended -> finalized -> archived
```

- `draft`: 내부 작성 중이며 외부에 노출하지 않는다.
- `scheduled`: 대회 공개 또는 시작 예약이 잡힌 상태이며 공개 목록과 공개 상세에는 노출하지 않는다.
- `open`: 대회 홈, 개요, 공지, 규정 등 공개 가능한 정보가 노출된다.
- `running`: 문제 열람과 제출이 가능하다.
- `ended`: 제출이 종료되고 대회 운영자/운영매니저 검토와 재채점 검토가 가능하다.
- `finalized`: 최종 결과가 확정되어 공식 스코어보드가 고정된다.
- `archived`: 종료 후 보관 상태이며 공개 정책에 따라 문제와 결과를 조회할 수 있다.

스코어보드 프리즈는 별도 대회 상태로 두지 않고 `running` 상태 안의 스코어보드 정책으로 관리한다.
프리즈 시작 기본값은 대회 종료 1시간 전이다.

- `scoreboard_freeze_enabled`
- `scoreboard_freeze_at`
- `scoreboard_unfreeze_at` 또는 수동 해제 여부

프리즈 중에도 대회 운영자와 권한 있는 대회 운영매니저는 실제 스코어보드를 확인할 수 있고, 참가자와 공개 사용자는 동결 정책이 적용된 스코어보드를 본다.
