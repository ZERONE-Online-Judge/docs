# 대회 규정

## 참가 규정

- 참가 단위는 팀이다.
- 대회 참가 단위는 항상 `n인 1팀`이며 `n >= 1`이다.
- 대회 운영진은 참가팀 등록 시 팀 이름, 팀장 이름/메일, 팀원 이름/메일을 등록한다.
- 참가자 로그인 식별자는 등록된 참가자 이메일이다.
- OTP는 참가자 이메일별로 발송한다.
- 로그인 세션과 동시 접속 제한은 팀 단위가 아니라 참가자 이메일 단위로 관리한다.

## 문제 및 제출 규정

- 대회 시작 후에만 참가 팀이 문제를 볼 수 있다.
- 한 문제는 C99, C++17, Python 3.13, Java 8 제출을 지원한다.
- 참가 팀은 자신의 제출을 볼 수 있다.
- 대회 운영자와 권한 있는 대회 운영매니저는 모든 팀의 제출 현황을 볼 수 있다.
- 대회 운영자와 권한 있는 대회 운영매니저는 공정성 검토와 생성형 AI 또는 외부 자동화 도구 사용 의심 사례 확인을 위해 모든 팀의 제출 소스코드를 볼 수 있어야 한다.

## 스코어보드 규정

- 대회는 명시적인 `채점 제도 유형`을 가져야 한다.
- 대회 관리자 페이지에서 대회별 점수 계산 방법을 설정할 수 있어야 한다.
- 기본 채점 제도는 `AC/WA 대회`와 `점수제 대회`로 구분한다.
- 세부 규정과 기본값은 [대회 채점 제도 구분](./contest-scoring-systems.md)을 따른다.

### AC/WA 대회 표준 규정

- 기존 기본 규칙은 BOJ 일반 대회 방식을 따른다.
- 부분 점수 문제는 사용하지 않는다.
- 정답으로 인정되는 결과는 `Accepted`만이다.
- `Accepted`를 제외한 모든 최종 판정 결과는 스코어보드에서 오답 처리한다.
- 페널티 계산에서는 `Compilation Error`를 제외한다.
- `Wrong Answer`, `Presentation Error`, `Runtime Error`, `Time Limit Exceeded`, `Memory Limit Exceeded`, `Output Limit Exceeded`, `System Error` 등은 페널티 가산 대상이다.
- 페널티 계산 기준 시각은 `submitted_at`이다.
- 이미 맞은 문제를 다시 맞아도 최초 accepted 제출 시각을 유지한다.
- 프리즈 시작 기본값은 대회 종료 1시간 전이다.
- 프리즈 이후 참가자와 공개 사용자는 프리즈 시점의 순위만 본다.
- 운영자 전용 상세 스코어보드는 실제 최신 점수를 보여준다.
- 종료 후 최종 공개는 자동으로 하지 않고 관리자가 명시적으로 수행한다.
- 종료 후 문제, 스코어보드, 제출 현황 공개 여부는 대회 설정에서 각각 제어한다.
- 비로그인 사용자는 대회 전과 대회 중에 문제, 스코어보드, 제출 현황을 볼 수 없다.
- 비로그인 공개 접근은 대회 종료 후 운영자가 해당 항목을 수동 공개한 경우에만 허용한다.
- 참가자는 로그인 후 팀에 등록된 참가 유형의 문제, 스코어보드, 제출 현황만 볼 수 있다.

### 채점 상태 및 최종 판정 결과

- `waiting`: 기다리는 중
- `preparing`: 채점 준비 중
- `judging`: 채점 중
- `accepted`: 맞았습니다
- `presentation_error`: 출력 형식이 잘못되었습니다
- `wrong_answer`: 틀렸습니다
- `time_limit_exceeded`: 시간 초과
- `memory_limit_exceeded`: 메모리 초과
- `output_limit_exceeded`: 출력 초과
- `runtime_error`: 런타임 에러
- `compile_error`: 컴파일 에러
- `system_error`: 시스템 에러

적용 원칙:

- `waiting`, `preparing`, `judging`는 진행 상태다.
- 최종 판정 결과 중 `accepted`만 정답 처리한다.
- 나머지 모든 최종 판정 결과는 스코어보드에서 오답 처리한다.
- `compile_error`는 페널티 가산 대상에서 제외한다.

### 점수제 대회 기본 규정

- 점수제 대회는 일반문제와 서브태스크형 문제를 함께 가질 수 있어야 한다.
- 모든 문제는 최대 `100점`을 가진다.
- 부분 점수가 없는 일반문제는 점수제 대회 안에서 `0점 또는 100점`만 나오는 문제로 처리한다.
- 문제별 배점 기준과 서브태스크 구성은 서로 다를 수 있다.
- 기본 스코어보드는 총점 기준으로 정렬한다.
- 기본 동점 처리는 해당 총점에 먼저 도달한 제출의 `submitted_at`이 빠른 팀을 상위에 둔다.
- 기본 점수 반영 방식은 채점이 완료되어 공식 반영되는 즉시 스코어보드에 적용하는 방식이다.
- 문제별 최고 점수만 반영한다.
- 새 제출 점수가 기존 최고 점수보다 높을 때만 점수와 점수 달성 제출 시각을 갱신한다.
- 새 제출 점수가 기존 최고 점수와 같거나 낮으면 기존 점수와 기존 달성 시각을 유지한다.
- 기본 동점 처리 기준까지 모두 같다면 중복 순위를 허용한다.

## 수정 규정

- 대회 진행 중에는 문제 본문과 데이터 수정이 기본적으로 금지된다.
- 수동 재채점은 제공하지 않는다.
- 대회 운영자/운영매니저와 서비스 마스터 모두 재채점을 요청할 수 없다.
- 재채점 결과 검토/반영 흐름은 운영 기능에서 제외한다.
- 대회 종료 후 관리자는 권한과 사유를 갖고 팀/문제별 점수를 개별 조정할 수 있다.
- 종료 후 수동 점수 조정은 감사 로그 대상이며, 최종 스코어보드에 반영할지는 운영자가 명시적으로 확정해야 한다.
