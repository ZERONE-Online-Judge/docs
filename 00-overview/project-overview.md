# 프로젝트 개요

## 목적

`Zerone Online Judge`는 일회성 사이트가 아니라 지속적으로 운영 가능한 온라인 저지 및 대회 플랫폼을 목표로 한다.
현재 단계에서는 구현보다 규정, 운영 정책, 권한, 플로우, 도메인 구조를 먼저 정리한다.

## 현재 개발 원칙

- 레거시는 보관용으로만 두고, 신규 디자인과 사용자 흐름은 새로 설계한다.
- 진행 순서는 `대회 규정/운영 정책 정리 -> DDD 정리 -> 구현`이다.
- 프론트는 `Vite` 기반 `React` 앱으로 개발하며, 스타일링에는 `Tailwind CSS`, 서버 상태 관리에는 `TanStack Query`, 클라이언트 전역 상태 관리에는 `Zustand`를 사용한다.
- 백엔드는 FastAPI를 기준으로 한다.
- 채점 시스템은 백엔드와 분리된 독립 계층으로 운영한다.

## 제품 목표

- 여러 대회를 장기적으로 운영할 수 있어야 한다.
- 대회 생성, 일정 설정, 종료, 운영 인력 배정이 가능해야 한다.
- 팀 참가, 문제 조회, 제출, 스코어보드 제공이 가능해야 한다.
- 대회 참가 단위는 항상 `n인 1팀`이며, `n >= 1`이다.
- 대회 성질에 따라 팀원 전원, 일부 팀원, 또는 단일 팀원만 접속 가능한 정책을 지원해야 한다.
- 채점 시스템은 FIFO, 확장성, 보안, 안정성을 최우선으로 가져야 한다.

## 문서 분리 기준

- 플랫폼 전반 흐름과 기술 요구사항은 `01-platform`
- 공개 페이지 플로우는 `02-public-pages`
- 권한 체계는 `03-permissions`
- 대회 단위 요구사항은 `04-contest`
- 대회 운영자 기능은 `05-operator`
- 서비스 마스터/서비스 관리 기능은 `06-service-admin`
- DDD와 내부 설계는 `07-architecture`
- 이전 통합 문서는 `99-archive`

## 현재 가장 먼저 정리할 것

- 대회 규정
- 팀 구성 및 로그인 정책
- 권한 모델
- 화면별 플로우
- 운영 단계별 금지/허용 범위
