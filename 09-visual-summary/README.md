# 사람이 보는 구현 요약

이 폴더는 백엔드 구현 전에 사람이 빠르게 합의하기 위한 요약 문서다.
상세 규칙은 기존 문서를 기준으로 하고, 여기서는 화면, API, 응답, 채점 흐름, 권한을 한눈에 보이게 정리한다.

## 읽는 순서

1. 어떤 화면이 있고 어떤 API가 필요한지 본다.
2. 제출이 들어간 뒤 백엔드와 채점기가 어떻게 움직이는지 본다.
3. 누가 어떤 운영 작업을 할 수 있는지 권한명을 확인한다.
4. 확정된 운영 정책과 남은 구현 전제를 확인한다.

## 구현 중 테스트 원칙

백엔드는 기능을 크게 완성한 뒤 한 번에 테스트하지 않는다.
기능 단위로 API, application service, domain rule 테스트를 중간중간 추가한다.

기능 하나의 완료 기준:

- 정상 API 요청이 테스트된다.
- 대표 실패 응답이 테스트된다.
- 권한 또는 scope가 필요한 API는 허용/거부 케이스가 테스트된다.
- 상태 전이가 있는 기능은 허용 전이와 금지 전이가 테스트된다.
- DB transaction이 중요한 기능은 rollback 또는 중복 처리 케이스가 테스트된다.
