# 페이지별 DDD 다이어그램

이 폴더는 화면/페이지 단위로 필요한 도메인 컨텍스트, 읽기 모델, 사용자 플로우, API 흐름을 정리한다.

## 문서 기준

- 제품 요구사항 문서는 `02-public-pages`, `04-contest`, `05-operator`, `06-service-admin`에 둔다.
- 도메인 전체 컨텍스트는 `07-architecture/ddd.md`를 기준으로 한다.
- 이 폴더는 페이지 구현 전에 프론트엔드와 백엔드가 공유해야 할 page-level DDD 흐름을 정리한다.
- 다이어그램은 Mermaid 문법을 사용한다.

## 문서 목록

- [root-public-pages.md](./root-public-pages.md): 메인/루트 공개 페이지 묶음
- [service-notice-pages.md](./service-notice-pages.md): 서비스 공지 게시판 공개/운영 페이지
- [contest-rules-pages.md](./contest-rules-pages.md): 대회 규정집 공개 페이지
- [contest-participant-pages.md](./contest-participant-pages.md): 대회 참가자 페이지
- [contest-operator-pages.md](./contest-operator-pages.md): 대회 운영자 페이지
- [public-judge-status-pages.md](./public-judge-status-pages.md): 공개 채점 서버 현황 페이지
- [staff-login-pages.md](./staff-login-pages.md): 서비스 관리자/권한자 로그인 페이지와 모달
- [service-admin-pages.md](./service-admin-pages.md): 서비스 관리자 페이지
- [service-admin-contest-pages.md](./service-admin-contest-pages.md): 서비스 관리자 대회 관리 페이지
- [service-manager-pages.md](./service-manager-pages.md): 서비스 매니저 관리 페이지
- [service-judge-pages.md](./service-judge-pages.md): 서비스 관리자 채점 인프라 페이지
- [service-mail-template-pages.md](./service-mail-template-pages.md): 서비스 관리자 메일 템플릿 페이지
